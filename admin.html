GitHub Pages는 서버가 없어서 OAuth 토큰 교환이 어렵다 보니, Fine-grained PAT(Contents: Read and write)로 로그인하는 방식이 가장 안정적이야.

⚠️ 토큰은 절대 레포/코드에 저장하지 말고, 이 페이지에서만 입력해. 공용 PC면 작업 후 로그아웃(토큰 삭제) 꼭!

관리

삭제는 GitHub에서 해당 md 파일을 삭제 커밋하는 방식.

※ GitHub Pages는 정적 호스팅이라 “진짜 서버 로그인”은 못 돌려. 대신 PAT로 GitHub API 커밋 권한을 가진 사람만 발행/삭제가 가능해.
